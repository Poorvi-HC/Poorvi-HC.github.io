Research
Integrated Cross-Organ Transcriptomic Analysis Uncovers Conserved Gene Signatures Predictive of Allograft Rejection
HC Poorvi, PK Vinod
(Preprint) BMC Genomics, 2025

This research establishes a unified, pan-organ molecular framework for allograft rejection through a network-based systems biology analysis of 672 transplant biopsies (heart, liver, kidney). We identified a conserved immune cascade and a novel 24-gene cell cycle signature, implicating cellular proliferation as a core pathological hallmark. A resulting 20-gene immune signature, derived via machine learning, demonstrates high-accuracy (AUC > 0.96) and robust, pan-organ diagnostic utility.

HyperCLSA: Hypergraph Contrastive Learning with Self-Attention for Multi-Omics Integration
Gaurav Bhole, HC Poorvi, JR Madhav, Prabhakar Bhimalapuram, PK Vinod
11th International Conference on Pattern Recognition and Machine Intelligence, 2025
bioRxiv

HyperCLSA is a novel deep learning framework for multi-omics data integration in cancer subtyping. It combines hypergraph-based sample encoding, supervised contrastive learning for latent space alignment, and multi-head self-attention for adaptive fusion of omics modalities.

Network File System (NFS)
C, TCP/IP Socket Programming, Multi-threading, Concurrency, Distributed Systems, LRU Cache

This project is a Network File System (NFS) built from scratch in C using TCP/IP socket programming. Its architecture features a central Naming Server that acts as a directory service, orchestrating communication between Clients and multiple Storage Servers. The system supports core distributed file operations (CREATE, DELETE, READ, WRITE, COPY), with the Naming Server managing metadata and Storage Servers handling physical storage. To optimize performance, the Naming Server implements an LRU cache for accelerating path lookups, while Storage Servers use semaphores and multi-threading to manage concurrent read/write operations.

Custom C-Shell
C, Linux System Calls, Process Management (fork/exec), Signal Handling, Job Control (fg/bg), IPC (Pipes), I/O Redirection, TCP/IP Sockets

This project is a custom C-Shell built entirely from scratch in C, featuring a highly modular architecture that separates command logic from core utilities. It fully implements essential shell functionalities, including robust process management (foreground/background execution via fork() and execvp()), job control (fg, bg), and custom signal handling for SIGCHLD, CTRL-Z, and CTRL-C. The shell correctly parses and executes commands involving multi-stage pipelines (|) and I/O redirection (<, >). It also features a wide range of custom built-in commands like peek (a feature-rich ls), warp (cd with history), pastevents (command history), seek (recursive search), and iMan, which fetches man pages from the internet by making a direct TCP socket connection.

Vocab Network
Python, NLTK (WordNet), NetworkX, igraph, Leiden Algorithm (for clustering), API Integration, JSON Caching

This project, the Vocabulary Network Generator, is a Python script that transforms a plain text word list into a data-rich, interactive network graph for studying. The script automatically enriches each word by fetching definitions and example sentences from an online dictionary API while using NLTK WordNet to map semantic relationships like synonyms and antonyms. It then intelligently groups related words by applying the Leiden algorithm for semantic clustering. For high-speed performance, all fetched data is stored in a local JSON cache, making subsequent runs nearly instantaneous. The final output is a .graphml file, ready for visualization in network analysis software like Cytoscape to explore the complex relationships between words.

Phylogenetic Tree Generation
Python, NumPy, Bioinformatics, UPGMA Algorithm, BLOSUM Matrix, Sequence Alignment

This Python project is a tool for understanding evolutionary relationships by automatically generating phylogenetic trees from biological data. It provides two distinct workflows to analyze either RNA (nucleotide) or protein sequences. The scripts work by first calculating the genetic similarity or distance between each sequence in an input file, and then using a clustering algorithm to progressively group the most closely related sequences. The final result is a phylogenetic tree printed directly to the terminal, offering a clear visual map of the evolutionary connections between the different sequences.
